Execution Layer — Tools
Docker

Sandbox

Ephemeral python:3.11-slim containers, no network access. Used by Engineer and QA.

Git

Git Tool

Branch, commit, and push operations. Controlled by Engineer and DevOps agents.

Linter

Linter Tool

Flake8 / Bandit static analysis. Used by the QA Agent.

Registry

Skills Registry

Dynamic plugin resolver for missing runtime dependencies.

Persistent Memory
Log

Decision Log

Immutable record of all CEO and CTO architectural decisions.

Artifacts

Artifact Store

Persists generated source code, design docs, and environment stubs.

Ledger

Cost Ledger

Centralized token usage tracking to prevent runaway loops.

Dashboard

Next.js Dashboard

Real-time UI via WebSocket from ai-org-events. Shows DAG viewer, live agent feed, and cost meter.

Observability

Prometheus + Logs

Prometheus metrics on port 8000. Structured JSON tracing via Structlog for production reliability.

Step-by-Step Request Lifecycle

Step Stage Component What Happens Key Files
01 Input User User submits a business idea, e.g. "Build a SaaS invoicing app" dashboard/
02 Gateway FastAPI HTTP POST or WebSocket message is validated and forwarded to the Orchestrator api/
03 Planning Orchestrator — Planner Planner calls Gemini 2.5 Flash to generate a task DAG from the business idea orchestrator/planner.py, task_graph.py
04 Dispatch Kafka Dispatcher Each task node is published to the ai-org-tasks Kafka topic orchestrator/kafka_dispatcher.py
05 Routing Rust MoE Engine Engine consumes tasks, scores against 6 agent experts in sub-milliseconds, routes to winner moe-scoring/, moe/router.py
06 Execution Agent (one of 6) Agent receives the task, calls Gemini, invokes tools, and produces structured output agents/base_agent.py, specific agent files
07 Tool Use Tools Layer Engineer / QA run code in Docker sandbox. DevOps uses Git. QA runs the linter. tools/docker_sandbox.py, git_tool.py, linter_tool.py
08 Result Kafka — Results Topic Agent publishes output to ai-org-results; Orchestrator updates the DAG node status messaging/kafka_client.py
09 Memory Persistence Layer Decision Log and Artifact Store are updated; Cost Ledger records token usage orchestrator/memory/
10 Observe Dashboard + Prometheus Events stream to Next.js dashboard via WebSocket; Prometheus scrapes metrics at port 8000 dashboard/, observability/
11 Loop / Done Orchestrator Orchestrator checks DAG — if more tasks remain, steps 4–10 repeat; otherwise user is notified orchestrator/task_graph.py
